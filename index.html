HTML

  • HTML stands for "HyperText Markup Language".
  • HTML provides strutcture and a basic framework to content on this webpage as well as other webpages and apps.
  • The structure is composed of many different elements, such as text elements, image elements, header elements, list elements, etc.
  • !DOCTYPE tells the browser what type of document to expect when it is accessed.
  • The head element contains information (also metadata - not visible to users) about the webpage, such as accessibility, search engines, and performance.
  • The body element represents the visible content shown to the user, such as lines of text, bulleted lists, levels of headers, line breaks, etc.

CSS

  • CSS stands for "Cascading Style Sheets".
  • CSS allows for the aesthetics of a webpage or app to be incorporated, such as color, grids, shading, etc., to better organize the webpage.
  • A margin indicates how much space we want around the outside of an element. Setting your margin as 0 means there will be no space around an element.
  • A padding indicates how much space we want around the content inside an element, and more padding mean enlarging an element. The inverse is also true.
  • There are three methods to style using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • When using external CSS style sheet, this needs to be linked to the HTML file.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git touch file/branch name: creates a new file in a designated folder.
  • git add -A: stages the files by adding all changes directly to current working area branch; do this after saving any changes in VS Code.
  • git commit -m "type message here": make a commit message; message must be in quotation marks.
  • git pull origin main: pulls the source code from GitHub to ensure that we have the latest commits and changes.
  • git push origin feature/branch name: pushes changes made to a specific branch to GitHub.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • A string is a letter or word contained in quotation marks. Numbers and Boolean values do not need quotation marks.
  • The command console.log() will output a message to the web console.
  • Control flow is the order in which a computer executes code in a script.
  • When working with a group of related or unrelated data, an array can be used to hold all of that data in a single variable.
  • The index indicates the location of an item within an array and numbers each item sequentially, beginning at 0.
  • The for loop executes a given block of code so long as a specified condition remains true.
  • To not delete code but render it inactive, type /* before and then */ after a block of code. This is called commenting out code.
